# 评审标准与评审视角

## 评审环境

- 全国年申请量超过45万项，竞争极其激烈
- 评审专家平均约**20分钟**看一份申请书
- 评审主体从小同行向大同行转变（5-7位通讯评审专家）
- 评审时间极端约束，申请书必须以高度科学逻辑的方式呈现

## 评审关注的核心要素

### 1. 科学问题的质量（最决定性因素）
- 问题是否独特且重要？
- 问题凝练是否清晰、聚焦？
- 是否避免了同质化和跟风？
- 是否是真正的科学问题（而非技术/工程问题）？

### 2. 创新性
- 是否有实质性创新（而非简单的工具移植）？
- 创新点是否明确且可区分？
- 研究内容 ≠ 创新点（不要混淆）

### 3. 逻辑性
- 立项依据 → 研究内容 → 研究方案，逻辑链是否完整？
- 是否符合科学逻辑的推演方式？
- 大同行专家是否能 follow 你的逻辑？

### 4. 可行性
- 研究方案是否具体、可操作？
- 申请人是否有能力完成？（"为什么是你？"）
- 是否有合理的风险应对措施？

### 5. 研究基础
- 前期工作积累是否与本项目相关？
- 代表作是否体现申请人的特色和能力？
- 已完成项目的执行情况如何？

### 6. 研究基础的"支撑性"检验（评审人视角）

评审人不只看"做过什么"，更看"前期工作能否逐项支撑研究内容"：

- **支撑映射是否清晰**：每块研究基础是否明确对应某项研究内容？评审人会逐条核对"研究内容3要做的验证，前期是否已跑通类似流程"。
- **三件套是否齐备**：论证文字、配图、代表作/专利是否成块出现？只有论文列表而无论证逻辑 = 成果堆砌（扣分）。
- **个人贡献是否可辨**：合作论文中申请人是否加粗、是否注明贡献（2026硬性要求）？无法辨识个人贡献的代表作几乎无效。
- **风险应对是否闭环**：是否识别了真正的核心风险，并给出"计算-实验-反馈"式可操作闭环，而非空泛保证？

### 7. 可行性的"背书性"检验（评审人视角）

- **每个可行性维度是否有证据**：数据/方法/实验/能力四维度是否各自落到"已有X"（已建数据库、已发表方法、已建平台、已主持项目）？无证据的"可行"等于没写。
- **方法可行性是否引用了已验证成果**：核心方法是否已在顶刊/评估集上验证？这是大同行最看重的"硬背书"。
- **工作条件是否落实**：是否明确依托的重点实验室、算力与实验平台，体现条件"已落实"而非"拟争取"？

> **给申请人的反推建议**：写完研究基础后，逐项自问"如果我是评审人，这块前期工作能让我相信对应的研究内容做得成吗？"——答不上来的块，要么补三件套，要么删。

## 常见扣分点

### 形式问题
- 格式不统一（参考文献格式、作者姓名顺序等）
- 错别字、语法错误
- 图片不清晰（从PDF截图）
- 中英文摘要不匹配
- 信息不准确（职称、成果标注等）

### 内容问题
- 立项依据不充分，逻辑链断裂
- 科学问题太大太空，不聚焦
- 创新点不明确或过多（"太多创新点 = 没有创新点"）
- 研究方案不具体，看不出研究对象和材料体系
- 夸大研究意义（"填补空白""世界首次""国际领先"）
- 遗漏重要的前人工作
- 同质化研究，缺乏独特性
- 研究内容之间缺乏逻辑关系（"东一榔头西一棒"）

### 态度问题
- 申请书过长、冗余（超过30页）
- 成果堆砌，不分逻辑
- 代表作与选题无关（只选高影响因子期刊）
- 团队组织包装痕迹重（拼凑型、碎片化）

## 评审人的心理

- 第一印象很重要：摘要和概念图决定评审人的初始态度
- 简洁流畅的申请书会留下"项目值得资助"的印象
- 语言拖沓、逻辑混乱会让专家失去耐心
- 低级错误暗示申请人不严谨、不认真
- 大同行评审时，科学逻辑比专业细节更重要

## 不予受理的常见原因

- 限项规定违反
- 申请条件不符
- 材料不完整
- 格式严重不规范
- 信息虚假

## 2026年政策变化要点

- 青C项目资助规模将加大
- 交叉科学部新增受理青C项目
- 申请书结构改革：正文分三大块（立项依据、研究内容、研究基础）
- 正文原则上不超过30页
- 代表作需说明个人贡献情况
- 同层次人才计划同一年度只能申请一项
- 重点项目牵头+承担合计限一项
- AI使用需声明和标识
