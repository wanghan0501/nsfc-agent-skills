# NSFC 申请书文献检索与引用生成

## 文献在申请书中的要求

- 参考文献约 **25~30 条**，以近 5 年为主
- CNS / 领域顶刊 / 顶会优先
- **国际文献为主** + 国内文献为辅 + 申请人自己的工作（高亮）
- 不要遗漏领域关键文献与"领域大牛"代表作（评审人会注意）
- 格式必须统一规范；有**当年文献**加分
- 立项依据可引用**国家/地方政策文件**，强化战略需求论证

---

## 文献组成配比清单（重要）

一份高质量申请书的参考文献不是"凑数 30 条"，而是有明确结构。
建议按以下配比组织（总量约 25~30 条，可按学科微调）：

| 文献类型 | 作用 | 大致占比 | 摆放位置 |
|----------|------|----------|----------|
| **领域奠基 / 高引经典** | 锚定研究脉络，证明你熟悉领域根基 | 25~35% | 立项依据综述段 |
| **近 1~2 年前沿**（含顶会论文） | 证明紧跟最新进展，凸显问题的"当下性" | 25~35% | 立项依据"现状与不足"段 |
| **本年度 / 投稿中 / 预印本** | 体现前沿敏感度（"有当年文献加分"） | 5~15% | 立项依据最新进展、研究基础 |
| **国家 / 地方政策文件**（EB/OL） | 论证国家战略需求与立项必要性 | 2~3 条 | 立项依据开篇 |
| **申请人自引代表作**（高亮） | 证明前期积累与本项目的强关联 | 4~8 条 | 正文 + 研究基础（加粗） |

**自查清单**：
- ✅ 是否有 2~3 条政策文件支撑"国家战略价值"？
- ✅ 是否有当年（本年度）发表或在投的文献？
- ✅ 自引代表作是否覆盖研究内容 1/2/3 各自的前期基础？
- ✅ 领域内"绕不开"的奠基工作有没有遗漏？
- ❌ 不要全是 5 年以上的老文献（显得脱离前沿）
- ❌ 不要只引自己人（显得领域视野窄）

## 文献检索

### OpenAlex API（免费，无需API key）

**脚本**：`scripts/search_literature.py`

**基本用法**：
```bash
# 搜索关键词，按引用数排序
uv run scripts/search_literature.py "machine learning potential" --limit 20 --sort cited_by_count

# 搜索近3年的文献
uv run scripts/search_literature.py "deep learning molecular dynamics" --year-from 2023

# 按发表日期排序（找最新文献）
uv run scripts/search_literature.py "neural network force field" --sort publication_date

# 紧凑输出（每篇一行）
uv run scripts/search_literature.py "density functional theory" --compact
```

**返回信息**：标题、作者、期刊、年份、DOI、引用数、摘要

**搜索策略建议**：
1. 先用宽泛关键词搜索了解领域全貌
2. 再用精确关键词搜索特定方向
3. 关注高引论文（领域经典）和最新论文（前沿动态）
4. 检查是否遗漏领域大牛的工作
5. 搜索中文关键词时可能需要用英文替代

## 引用生成

### wenxian

[wenxian](https://github.com/njzjz/wenxian/tree/master/skill) 是一个学术引用生成工具，支持从 DOI、PMID、arXiv ID 等标识符生成标准引用格式。

**脚本**：`scripts/generate_references.py`

**用法**：
```bash
# 从DOI列表文件生成BibTeX引用
uv run scripts/generate_references.py refs.txt --format bibtex

# 生成纯文本引用
uv run scripts/generate_references.py refs.txt --format text

# 输出到文件
uv run scripts/generate_references.py refs.txt --format bibtex --output refs.bib
```

**refs.txt 格式**（每行一个标识符）：
```
10.1103/PhysRevLett.120.143001
10.1038/s41586-020-2242-8
arXiv:2304.09423
```

### NSFC引用格式

NSFC申请书通常使用**编号引用格式** [1], [2], ...

**格式示例**：
```
[1] 作者1, 作者2, 等. 标题. 期刊, 卷(期): 起始页-结束页, 年份.
[2] Author1, Author2, et al. Title. Journal, Volume(Issue): Pages, Year.
```

**注意**：
- 所有文献格式必须统一（姓名顺序、期刊缩写等）
- 建议手动录入基金委成果系统保持格式一致
- 自己的文献可以**加粗**或用其他方式高亮

### 特殊文献类型著录格式

申请书文献不止期刊论文。以下三类**最易写错、最易被技能漏掉**（均为虚构占位示例）：

**1. 政策文件 [EB/OL]**（立项依据论证国家战略时使用）
```
[1] 机构A, 机构B, 等. 某领域发展规划[EB/OL]. 2024, https://example.gov.cn/xxx.htm.
[2] 某部委. 关于推动某产业高质量发展的意见[EB/OL]. 2025, https://example.gov.cn/yyy.htm.
```
- 著录类型用 `[EB/OL]`（电子/网络资源）
- 含机构作者、文件名、发布年份、URL
- ⚠️ 链接需真实可访问；提交前逐条点开验证（避免"图片/链接打不开"类不予受理风险）

**2. 预印本 / 投稿中 [J]**（体现前沿性，含 bioRxiv / arXiv / Early Access）
```
[3] 作者A, 作者B, 等. 某方法的标题[J]. bioRxiv, 20XX: 20XX.MM.DD.XXXXXX.（某顶刊审稿中）
[4] 作者C, 等. 某算法的标题[J]. arXiv preprint arXiv:XXXX.XXXXX, 20XX.
[5] 作者D, 等. 某模型的标题[J]. 某顶刊, 2026. Early Access.
```
- bioRxiv/medRxiv：用其投稿号作"页码位"，可在括号注明"（XX审稿中）"
- arXiv：用 `arXiv preprint arXiv:编号, 年份` 标准写法
- 期刊已接收未排版：标 `Early Access` / `在线发表`
- ⚠️ 投稿中论文真实性靠人工把握，AI 检索不到——这正是不能让 AI 代笔的原因之一

**3. 会议论文 [C] 与专利 [P]**（计算/工程学科常见）
```
[6] 作者E, 等. 某模型的标题[C]//某国际会议(简称 年份). 出版方, 年份: 页码.
[7] 发明人A, 发明人B, 等. 某方法及装置: 地区, 专利号[P]. 授权/公开日期.
```
- 会议论文用 `[C]//会议全称`；与期刊 `[J]` 混排时类型标识务必正确
- 专利用 `[P]`，注明专利号与状态（已授权 / 公开）

### 自引代表作高亮规范

申请人自己的工作要在**两处**出现，处理方式不同：

| 位置 | 写法 | 示例（脱敏） |
|------|------|--------------|
| **正文论述中** | 以上标编号 `[n]` 融入行文，说明"申请人前期开发了 X 方法" | "……申请人前期提出了某方法[19]，初步验证了……" |
| **研究基础整条列出** | 整条著录，**申请人姓名加粗**，括注期刊与状态 | "**作者A（申请人）**, 作者B, 等. 标题[J]. 某顶刊, 2024."（加粗为项目申请人） |

**规范要点**：
- 研究基础列表统一注明"（加粗为项目申请人）"
- 共同一作用 `#`、通讯作者用 `*` 标注（与代表作个人贡献声明呼应，见 nsfc-policy）
- 每篇自引代表作应在正文/研究基础中**点明支撑哪一项研究内容**

## 完整工作流程

```
1. 确定研究方向和关键词
        ↓
2. 用 search_literature.py 搜索相关文献
        ↓
3. 按"文献组成配比清单"筛选 ~25-30 条
   - 政策文件(2-3条) + 领域奠基 + 近1-2年前沿 + 本年度/在投 + 自引代表作
   - 政策/预印本/会议论文需手动补著录（工具难自动生成）
        ↓
4. 收集所有 DOI，写入 refs.txt
        ↓
5. 用 generate_references.py 生成标准引用
        ↓
6. 检查格式统一性
        ↓
7. 确认没有遗漏关键文献
```

## 常见问题

### 搜索不到文献？
- 尝试不同的关键词组合
- 使用英文关键词
- 检查拼写
- OpenAlex覆盖面广但不是100%，可辅以Google Scholar

### wenxian报错？
- 确认 `uvx` 已安装（`uv` 的一部分）
- 确认DOI格式正确（如 `10.1038/s41586-020-2242-8`）
- 部分API可能需要网络代理，请根据本地环境配置

### 格式不统一？
- 使用同一工具生成所有引用
- 手动检查并统一姓名顺序、期刊缩写
- 建议最后统一校对一遍
